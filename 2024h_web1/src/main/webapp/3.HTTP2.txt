[ HTTP ] HyperText Transfer portocol
		- 문자 이동 프로토콜 : 객체지향 아니므로 문자 자료만 통신이 가능하다.
		1. 정의 : 웹에서 사용하는 문자 통신 규약
		2. 특징
			1. 상태가 무상태 프로토콜 (상태가 없고 -> 저장기능이없다 휘발성)
			2. 독립적, 이전 요청과 다음 요청은 다르다
			3. TCP/IP 프로토콜에 종속되어 동작 PORT 80/808 사용한다.
		3. 주소체계
			URL : 웹 주소, 네트워크 상의 자료가 어디에 있는지 위치 규약
				http://ip:port/자료위치
				http://nid.naver.com/nidlogin.login :네이버의 login(html) 이 위치한 주소
				
				
			URI : 각종 자료들의 유일한 이름이나 위치를 표시하는 규약 , URL + 매개변수
				http://ip/port/자료위치?매개변수1=값1&매개변수2=값2
				
				https://finance.naver.com/item/main.naver?code=005930
				(URL) https://finance.naver.com/item/main.naver
				(URI) https://finance.naver.com/item/main.naver?code=005930
				
		4. HTTP Header (HTTP 통신정보)
			1. HTTP ***METHOD*** : http 이용한 통신 요청할때 분류하는 함수
			2. HTTP ***BODY***	 : HTTP 이용하여 데이터의 내용/본문 보낼때 사용되는 규칙
			3. HTTP Content-Type : HTTP 이용한 데이터의 타입 정보
			4. HTTP HOST	     : HTTP 이용한 통신 요청하는 서버의 URL 주소
			
		5. HTTP METHOD 종류			CRUD					HTTP기반의 연결(매핑) 구축(RESTFUL)		httpServlet
			1. get					:R :read				get 매핑								doGet()
			2. post					:C :create				posts매핑								doPost()
			3. put					:U : update				put매핑								doPut()
			4. delete				:D : delet				delete매핑							doDelete()
			
			
		6. HTTP 사용처 :
			1. 브라우저(크롬,엣지,사파리 등등) 상단에 HTTP 입력한다. http get method 만지원
			2. Talend Api (restful test)
