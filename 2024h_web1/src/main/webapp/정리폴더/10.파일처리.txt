[ 입출력 ]
	- 입력 : 자료가 컴퓨터 (키보드 등등, 자바/프로그램)안으로
	- 출력 : 자료가 컴퓨터 (모니터 등등, 파일)밖으로 
[ 파일 ]
	- 컴퓨터가 자료들을 모아놓은 것, 
	- 윈도우 기준의 파일은 최소 용량 단위는 바이트 
[ 스트림 ]
	1. 정의 : 컴퓨터내 데이터/자료 가 다니는 데이터 통로
	2. 데이터 : *바이트* 단위의 데이터들이 스트림 통로로 데이터를 주고 받는다
	3. 사용처 : (프로그램 - 키보드), (프로그램 - 파일), (프로그램 - 네트워크) 등등
[ 바이트 ]
	1. 정의 : 컴퓨터가 자료를 저장하는 용량 단위
	2. 종류 : 
		1. bit : 0 또는 1
		2. byte : 01010101 (8bit -> 1byte)
		3. bit -> byte -> kb -> mb -> gb -> 등등
[ 자바 파일 입출력 ]
	1. 정의 : 자바에서 저장된 메모리(JVM)을 자바 외부인 파일(윈도우)에 입출력 하기.
	* 코드파일이 저장된 메모리와 실행중인 메모리는 다르다.
		코드는 하드에 저장되고 코드 실행은 메모리에 저장된다.
	2. 목적 : 실행중인 자바(JVM/메모리) 데이터를 외부(c:드라이브)에 저장하는 방법
		* 저장하는 방법 다양 : DB vs 파일처리 vs 톰캣세션 vs 브라우저세션 vs 자바JVM
	3. 클래스	
		1. FileOutputStream
			1. .write()
		2. FileInputStream	
			1. .read();
			
	* String 클래스 메소드
		.getBytes() : 문자열  --> 바이트배열로 반환 함수.
		new String(바이트배열) : 바이트배열 --> 문자열로 객체 생성
				
[ 자바 파일 클래스 ]
	- 자바에서 파일을 다루는 여러가지 메소드를 제공하는 클래스
	- 메소드
		* File file = new File('파일경로');
		1. .isFile()	: 지정된 파일이 존재하는지 반환 함수,  true/false
		2. .getName()	: 지정된 파일 이름 반환 메소드
		3. .exists()	: 파일 존재 여부 반환 메소드
		4. .length()	: 파일 용량(바이트) 반환 메소드
		5. .delete()	: 지정된 파일이 존재할 경우 삭제 메소드
		6. .getPath()	: 지정된 파일의 경로 반환 메소드
		7. .mkdir()		: 지정된 경로 폴더 생성 메소드
		8. .exists()	: 존재하는지 확인 메소드
